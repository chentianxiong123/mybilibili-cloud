## 1. 当前目标

当前主线不是单一前端项目，而是一个微服务仓库，里面混合了 Java 后端、多个独立前端、剪辑工作室和 OBS 桌面端。

核心目标:

- 后端继续保持 Spring Cloud 微服务架构。
- 前端按独立服务维护，不在根目录强行合并成一个 pnpm workspace。
- 包管理统一使用 pnpm，但每个前端服务各自独立安装、独立 lock、独立启动。
- 桌面端方向是本地 OBS 壳 + mybilibili 直播接入，砍掉 Streamlabs、多平台推流、商店、Prime/Ultra、Widget 等不融合业务。
- 界面方向保留中文、白色主题、mybilibili 自有直播业务，不保留外部平台业务。
- 后续所有删除操作必须先逻辑删除，不能直接物理删除业务目录。

## 4. pnpm 约定

当前约定:

- 根目录不做统一 pnpm workspace。
- 每个前端服务自己有 `package.json`、`.npmrc`、`pnpm-lock.yaml`。
- pnpm 全局 store 使用 `D:\.pnpm-store`。
- 每个服务目录内执行 `pnpm install` / `pnpm dev` / `pnpm build`。

推荐命令:

```powershell
pnpm config set store-dir "D:\.pnpm-store\v11"
pnpm store path
```

用户端:

```powershell
cd D:\files\mybilibili-next\mybilibili-cloud\mybilibili-web
pnpm install
pnpm dev
```

管理端:

```powershell
cd D:\files\mybilibili-next\mybilibili-cloud\mybilibili-admin-web
pnpm install
pnpm dev
```

WAP:

```powershell
cd D:\files\mybilibili-next\mybilibili-cloud\mybilibili-wap
pnpm install
pnpm dev
```

剪辑工作室:

```powershell
cd D:\files\mybilibili-next\mybilibili-cloud\mybilibili-studio-web
pnpm install
pnpm dev
```

pnpm 11 如果报 `ERR_PNPM_IGNORED_BUILDS`:

```powershell
pnpm approve-builds
```

常见需要审批的包:

- `esbuild`
- `@parcel/watcher`
- `vue-demi`

注意:

- `approve-builds` 是项目级别，不是全局一次解决。
- 如果不想交互，后续可以在各项目 `package.json` 固化 `pnpm.onlyBuiltDependencies`。

## 9. 删除安全规则

这是本仓库后续必须遵守的规则:

- 不直接删除业务目录。
- 不把 untracked 当垃圾。
- 不用 `git reset --hard`、`git clean -fdx`。
- 不在无人值守状态下执行递归物理删除。
- 删除前先列清单，并说明每一类删除理由。
- 删除动作改成移动到 `.trash/`。

推荐逻辑删除方式:

```powershell
$stamp = Get-Date -Format "yyyyMMdd-HHmmss"
$trash = Join-Path (Get-Location) ".trash\$stamp"
New-Item -ItemType Directory -Path $trash -Force | Out-Null
Move-Item -LiteralPath ".\待删除目录" -Destination $trash
```

`.trash/` 应加入 `.gitignore`，由人工定期物理清理。
