# 微服务项目搭建指导文档

## 一、项目概述

### 1.1 项目背景
基于现有的在线学习视频社区平台单体架构，拆分为微服务架构。原项目保留，新项目独立开发。

### 1.2 项目名称
- 原项目：mybilibili（单体架构）
- 新项目：mybilibili-cloud（微服务架构）

### 1.3 服务拆分
拆分为8个核心微服务：

| 服务名 | 端口 | 职责 |
|--------|------|------|
| mybilibili-gateway | 8080 | API网关，路由转发，认证鉴权 |
| mybilibili-user | 8081 | 用户注册登录、个人信息、关注 |
| mybilibili-video | 8082 | 视频上传、转码、播放、稿件管理 |
| mybilibili-danmaku | 8083 | 弹幕发送、加载、管理 |
| mybilibili-search | 8084 | 全文检索、热搜榜、搜索建议 |
| mybilibili-comment | 8085 | 评论发布、回复、管理 |
| mybilibili-interaction | 8086 | 点赞、投币、收藏、分享 |
| mybilibili-message | 8087 | 私信、通知、消息管理 |
| mybilibili-ai | 8088 | 字幕生成、AI摘要 |

## 二、技术栈

### 2.1 核心框架
- Spring Boot 2.7.x
- Spring Cloud 2021.x
- Spring Cloud Alibaba 2021.x

### 2.2 微服务组件
| 组件 | 版本 | 用途 |
|------|------|------|
| Nacos | 2.2.x | 服务注册与配置中心 |
| Spring Cloud Gateway | 3.1.x | API网关 |
| OpenFeign | 3.1.x | 服务间调用 |
| Sentinel | 1.8.x | 流量控制（可选） |

### 2.3 数据存储
| 组件 | 版本 | 用途 |
|------|------|------|
| MySQL | 8.0 | 关系型数据库 |
| MongoDB | 6.0 | 文档型数据库（弹幕、字幕） |
| Redis | 7.0 | 缓存、会话、排行榜 |
| Elasticsearch | 8.x | 全文检索 |

### 2.4 中间件
| 组件 | 版本 | 用途 |
|------|------|------|
| RabbitMQ | 3.12 | 消息队列（异步任务） |
| FFmpeg | 6.0 | 视频转码、音频提取 |

## 八、服务间调用

### 8.1 Feign客户端定义

```java
package com.mybilibili.user.feign;

import com.mybilibili.common.vo.Result;
import com.mybilibili.common.vo.UserVO;
import org.springframework.cloud.openfeign.FeignClient;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;

@FeignClient(name = "mybilibili-user", path = "/user")
public interface UserClient {
    
    @GetMapping("/{id}")
    Result<UserVO> getUserById(@PathVariable Integer id);
}
```

### 8.2 Feign客户端使用

```java
@Service
public class VideoServiceImpl {
    
    @Autowired
    private UserClient userClient;
    
    public VideoDetailVO getVideoDetail(Integer videoId) {
        // 获取视频信息
        Video video = videoMapper.selectById(videoId);
        
        // 调用用户服务获取创作者信息
        Result<UserVO> result = userClient.getUserById(video.getUserId());
        UserVO user = result.getData();
        
        // 组装返回
        VideoDetailVO vo = new VideoDetailVO();
        vo.setVideo(video);
        vo.setUser(user);
        return vo;
    }
}
```

## 十、代码迁移指南

### 10.1 迁移步骤

1. **复制公共模块**
   - 从原项目复制 `entity/`、`dto/`、`vo/`、`utils/` 到 `mybilibili-common`

2. **按服务拆分代码**
   - Controller → 对应服务的 `controller/`
   - Service → 对应服务的 `service/`
   - Mapper → 对应服务的 `mapper/`

3. **修改包名**
   - 原：`com.mybilibili.web`
   - 新：`com.mybilibili.user`（按服务名）

4. **添加Feign客户端**
   - 跨服务调用改为Feign接口

### 10.2 迁移对照表

| 原项目代码位置 | 新项目位置 |
|---------------|-----------|
| `mybilibili-common/entity/User.java` | `mybilibili-common/entity/User.java` |
| `mybilibili-web/controller/UserController.java` | `mybilibili-user/controller/UserController.java` |
| `mybilibili-web/service/UserService.java` | `mybilibili-user/service/UserService.java` |
| `mybilibili-web/mapper/UserMapper.java` | `mybilibili-user/mapper/UserMapper.java` |

## 十二、注意事项

1. **数据库连接**：各服务可共用一个数据库，也可拆分为多个数据库
2. **JWT认证**：网关统一验证，用户ID通过请求头传递给下游服务
3. **跨服务事务**：暂时不引入Seata，后续可添加
4. **日志追踪**：建议添加Sleuth+Zipkin实现链路追踪
5. **配置管理**：使用Nacos配置中心统一管理各服务配置

1. 先搭建父项目和公共模块
2. 搭建Gateway网关

4. 逐个搭建其他服务，迁移对应代码
5. 添加Feign客户端，实现服务间调用
