# 用户边界权限管理功能实现计划

## 功能概述

实现用户个人资料页的边界权限管理，包括：
1. 防止用户修改他人的个人简介和公告
2. 根据关注状态显示不同的代表作视图
3. 只显示已上架的稿件
4. 限制"TA的视频"显示数量为6个

## 需求分析

### 1. 用户边界权限控制
- **当前问题**：所有用户都可以编辑个人简介和公告
- **目标**：只有空间主人才能编辑自己的简介和公告
- **实现方式**：使用已有的`isOwnSpace`计算属性控制编辑权限

### 2. 关注状态视图切换
- **当前问题**：代表作视图切换按钮需要手动点击
- **目标**：根据关注状态自动切换视图
  - 已关注用户：显示"我的粉丝会看到"（显示1个视频）
  - 未关注用户：显示"新访客会看到"（显示3个视频）
- **实现方式**：使用`isFollowing`状态自动设置`representativeViewMode`

### 3. 只显示已上架稿件
- **当前问题**：显示所有状态的稿件
- **目标**：只显示status=3（已上架）的稿件
- **实现方式**：在API调用时添加status=3参数

### 4. 限制TA的视频显示数量
- **当前问题**：显示所有视频
- **目标**：主页"TA的视频"区域只显示6个视频
- **实现方式**：在前端对`allVideos`进行slice限制

### API修改（index.js）

#### 修改getVideosByUserId接口
**位置**：第121行

**修改内容**：
```javascript
getVideosByUserId: (id, sort, status) => {
  let url = `/manuscript/user/${id}`
  const params = []
  if (sort) params.push(`sort=${sort}`)
  if (status !== undefined) params.push(`status=${status}`)
  if (params.length > 0) url += `?${params.join('&')}`
  return api.get(url)
}
```

1. **向后兼容性**

2. **用户体验**
