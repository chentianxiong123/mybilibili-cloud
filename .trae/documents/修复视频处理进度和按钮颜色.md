# 修复视频处理页面进度条和流水线按钮颜色问题

## 问题分析

### 问题1：进度条没有连贯显示
- 现状：进度条只在收到SSE消息时更新，没有模拟进度
- 要求：前端自己做假进度条，一直加到99%卡住，SSE完成时瞬间跳到100%

### 问题2：流水线按钮颜色不正确
- 现状：完成全部任务后，只有AI是绿色，前面三个是灰色
- 原因分析：
  1. `processingStatusMap` 只映射了 `transcode`, `audio`, `subtitle`, `ai`
  2. SSE发送的 stage 包含 `transcode_success`, `audio_success`, `subtitle_success`, `ai_success` 等
  3. 完成状态判断 `stage === 'transcode'` 不会匹配 `transcode_success`
  4. 导致完成状态的更新被跳过

## SSE消息中的stage值分析

从日志中可以看到的stage值：
- `transcode` - 转码中
- `transcode_success` - 转码成功
- `audio_extracting` - 音频提取中
- `audio_success` - 音频成功
- `subtitle_generating` - 字幕生成中
- `subtitle` - 字幕处理中
- `subtitle_success` - 字幕成功
- `ai_summarizing` - AI摘要中
- `ai` - AI处理中
- `ai_success` - AI成功
- `completed` - 全部完成

## 修复方案

### 1. 修复进度条模拟逻辑

修改 `handleSseMessage` 函数：
- 收到任何处理中的消息时，启动模拟进度
- 模拟进度自动增长到99%后卡住
- 收到完成消息时，瞬间跳到100%

### 2. 修复流水线按钮颜色

修改 `handleSseMessage` 函数中的状态映射：

```javascript
// 处理中状态映射
const processingStatusMap = {
  'transcode': 1,
  'transcode_success': 11,  // 转码完成
  'audio_extracting': 2,
  'audio': 2,
  'audio_success': 21,      // 音频完成
  'subtitle_generating': 3,
  'subtitle': 3,
  'subtitle_success': 31,   // 字幕完成
  'ai_summarizing': 4,
  'ai': 4,
  'ai_success': 5,          // AI完成（全部完成）
  'completed': 5            // 全部完成
}
```

### 3. 优化进度显示逻辑

- 当收到 `*_success` 或 `completed` stage 时，直接更新表格中的 `processStatus`
- 不再依赖 `progress === 100` 判断完成

## 实施步骤

### Step 2: 优化模拟进度逻辑
- 确保模拟进度在99%卡住
