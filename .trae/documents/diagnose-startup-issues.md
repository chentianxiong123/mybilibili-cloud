# 项目启动不稳定问题诊断计划

## 问题描述
每次启动项目都会出现各种 500/404 错误，即使之前已经修复过。

## 当前错误分析

### 1. 500 Internal Server Error（服务端内部错误）
- `/api/manuscript/favorite/folders` - 收藏夹接口
- `/api/watch-history` - 历史记录接口
- `/api/collection/user/4` - 合集接口

**可能原因：**
- 后端服务未完全启动
- 数据库连接失败
- Nacos 注册中心连接失败
- 服务间调用失败

### 2. 404 Not Found（接口不存在）
- `/api/user/privacy/settings` - 隐私设置接口

**可能原因：**
- 接口未实现
- Gateway 路由配置缺失
- 服务未注册到 Nacos

## 根本原因分析

### 为什么每次启动都不稳定？

1. **启动顺序问题**
   - 微服务架构需要按顺序启动：Nacos → MySQL/Redis → 各微服务 → Gateway
   - 如果顺序不对，服务注册/发现会失败

2. **服务依赖问题**
   - 服务 A 依赖服务 B，如果 B 还没启动完成，A 的请求会失败
   - 没有实现优雅启动/健康检查

3. **数据库连接问题**
   - MySQL 连接池初始化失败
   - Redis 连接失败

4. **Nacos 注册延迟**
   - 服务启动后需要时间注册到 Nacos
   - Gateway 需要时间发现新注册的服务

5. **代码未保存/未编译**
   - 昨天的修改可能没有保存
   - 或者没有重新编译

## 诊断步骤

### 第一步：检查基础服务状态
1. 检查 Nacos 是否正常运行
2. 检查 MySQL 是否正常运行
3. 检查 Redis 是否正常运行

### 第二步：检查后端服务启动状态
1. 检查各微服务是否成功启动
2. 检查各微服务是否成功注册到 Nacos
3. 检查 Gateway 路由配置

### 第三步：检查后端日志
1. 查看 500 错误的具体堆栈信息
2. 确认是数据库问题还是服务调用问题

### 第四步：验证修复是否保存
1. 检查昨天修改的代码是否还在
2. 检查是否需要重新编译

## 建议的解决方案

### 短期方案（立即修复）
1. 确保按正确顺序启动服务
2. 等待所有服务完全启动后再测试
3. 检查并修复具体错误

### 长期方案（提高稳定性）
1. 添加健康检查端点
2. 实现服务优雅启动
3. 添加启动依赖检查
4. 使用 Docker Compose 统一管理启动顺序
