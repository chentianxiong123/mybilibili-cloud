# 前端模拟进度条方案

## 需求概述
前端直接使用模拟数据来显示进度条，不同任务类型有不同的进度增长速度，当进度达到90%后变慢，后端完成任务时通过SSE直接将进度变为100%并触发按钮变绿。

## 进度增长规则
| 任务类型 | 进度增量 | 说明 |
|---------|---------|------|
| 转码 | 1% | 较慢增长 |
| 字幕 | 1% | 较慢增长 |
| 音频 | 2% | 中等增长 |
| AI字幕 | 2% | 中等增长 |

**90%后变慢规则**：当进度达到90%后，增量减半，间隔时间加倍

## 实施步骤

### 1. 创建进度模拟器工具类
**文件**: `mybilibili-web/src/utils/progressSimulator.js`

创建一个通用的进度模拟器类，支持：
- 配置不同的进度增量
- 配置不同的更新间隔
- 90%后自动变慢
- 支持暂停/恢复/重置
- 支持外部强制完成（跳到100%）

```javascript
// 核心逻辑
class ProgressSimulator {
  constructor(options) {
    this.increment = options.increment || 1      // 进度增量
    this.interval = options.interval || 500      // 更新间隔(ms)
    this.slowDownAt = options.slowDownAt || 90   // 变慢阈值
    this.currentProgress = 0
    this.isRunning = false
    this.isCompleted = false
  }
  
  // 开始模拟
  start(callback) { ... }
  
  // 强制完成（SSE收到完成消息时调用）
  forceComplete() { ... }
  
  // 暂停/恢复/重置
  pause() / resume() / reset() { ... }
}
```

## 进度模拟器详细设计

### 状态机
```
[空闲] --start--> [运行中] --forceComplete--> [已完成]
   ^                  |                            |
   |                  |                            |
   +----reset---------+--------reset---------------+
```

### 进度计算逻辑
```javascript
calculateNextProgress() {
  if (this.currentProgress >= this.slowDownAt) {
    // 90%后变慢：增量减半，间隔加倍
    return {
      increment: this.increment / 2,
      interval: this.interval * 2
    }
  }
  return {
    increment: this.increment,
    interval: this.interval
  }
}
```

### SSE完成事件处理
```javascript
// 收到后端完成事件时
onSSEComplete(videoId) {
  const simulator = this.simulators.get(videoId)
  if (simulator) {
    simulator.forceComplete()  // 直接跳到100%
    // 触发按钮变绿
    this.updateButtonStatus(videoId, 'completed')
  }
}
```

## UI效果

### 进度条显示
- 0-89%: 正常速度增长
- 90-99%: 变慢增长（模拟等待后端完成）
- 100%: 收到SSE完成消息后直接跳转

### 按钮状态
- 处理中: 灰色/禁用状态
- 完成: 绿色/可点击状态
